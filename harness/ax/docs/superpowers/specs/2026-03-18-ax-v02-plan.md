# ax v0.2 — 시각화 중심 프로젝트 개선 계획

> 날짜: 2026-03-18
> 이전 이름: harness1 → **ax**
> 핵심 방향: 시각화 품질을 압도적으로 높이는 에이전트 팀 자동 생성

---

## 1. 현황 요약

### 이번 슬라이드 실행에서 드러난 문제

| 문제 | 영향 | 심각도 |
|------|------|--------|
| 네이밍/경로 불일치 (`harness` vs `harness1`) | 템플릿 참조 실패 위험 | HIGH |
| 커맨드-스킬 407줄 중복 | 유지보수 부담 | HIGH |
| 기존 프로젝트에 에이전트 추가 불가 | 실용성 저하 | HIGH |
| 권한 승인 인터럽트 6회+ | UX 저하 | HIGH |
| 피드백 루프/진행 추적 없음 | 자동화 부족 | MEDIUM |
| 비코드 도메인에서 불필요한 코드 탐색 | 시간 낭비 | MEDIUM |
| 생성된 팀에 시각화 전문가 없음 | 결과물 품질 저하 | CRITICAL |

### 7개 리서치에서 도출된 핵심 인사이트

1. **AI 슬롭의 근본 원인**: "분포 수렴" — 제약 없이 생성하면 Inter폰트+보라색 그라디언트로 수렴 (Anthropic 확인)
2. **극복법**: 디자인 시스템을 사전 주입하고, 제약 기반으로 생성 → 평가 → 재생성 루프
3. **3개 시각화 에이전트**: visual-architect(토큰 설계), visual-builder(차트/이미지 생성), visual-qa(정량 검증)
4. **멀티 LLM**: Claude(오케스트레이션) + OpenAI(이미지 생성) + Gemini Vision(품질 검증)
5. **정량 평가 루브릭**: 60점 만점 6축 평가, 45점 이상이면 PASS
6. **비용**: 이미지 포함 프레젠테이션 1회 $3-5, Gemini 무료 티어로 개발 가능

## 2. ax v0.2 아키텍처

### 2.1 파이프라인 (기존 6단계 → 8단계)

```
Phase 0: 초기화 (이름 파싱, 디렉토리, 권한 자동 설정)
Phase 1: 도메인 분석 (코드/비코드 분기)
Phase 2: 팀 아키텍처 설계 (패턴 선택 + 시각화 에이전트 기본 포함)
Phase 3: 에이전트 정의 생성
Phase 4: 스킬 생성
Phase 5: 오케스트레이터 생성 (CLAUDE.md)
Phase 6: 검증 & 테스트 (구조 + 시나리오)
Phase 7: [신규] 실행 (--execute 시 생성된 팀으로 도메인 작업 수행)
```

### 2.2 시각화 에이전트 3인조 (모든 팀에 기본 포함)

```
visual-architect (haiku) ──→ 디자인 토큰 파일 생성 (1회)
       │
       ▼
visual-builder (sonnet) ──→ 차트/아이콘/AI이미지 생성 (반복)
       │
       ▼
visual-qa (sonnet) ──→ 정량 기준으로 검증 (반복)
       │
       ├─ PASS (45/60+) → 완료
       └─ FAIL → visual-builder에 피드백 (최대 2회)
```

### 2.3 멀티 LLM 통합

```
Claude (ax 오케스트레이터)
  ├─ 콘텐츠 구조 + 코드 생성 (HTML/CSS/SVG)
  ├─ 이미지 프롬프트 생성
  └─ 최종 통합

OpenAI gpt-image-1.5 (이미지 생성)
  ├─ 아이소메트릭 일러스트
  ├─ 추상 배경 패턴
  └─ 커스텀 아이콘

Gemini Vision (품질 검증)
  ├─ Playwright 스크린샷 → 시각 품질 평가
  └─ 구체적 수정 지침 생성

폴백 체인:
  API 키 있음 → 해당 LLM 사용
  API 키 없음 → 순수 SVG/CSS 폴백 (외부 의존성 0)
```

## 4. 시각 품질 평가 루브릭 (60점 만점)

리서치에서 도출된 정량 기준. visual-qa 에이전트가 사용.

| 축 | 만점 | 기준 |
|----|------|------|
| **색상 조화** | 10 | 60-30-10 규칙 준수, CSS 변수 100%, 하드코딩 색상 0개, WCAG AA 4.5:1+ |
| **타이포그래피** | 10 | H1/Body 비율 2.5~5:1, 폰트 2개 이하, 시스템 폰트 아닌 의도적 선택, 행간 1.4~1.6 |
| **레이아웃/공간** | 10 | 그리드 정렬 일관, 여백 30%+, 연속 동일 레이아웃 2회 이하, 시각적 위계 명확 |
| **데이터 시각화** | 10 | 데이터-잉크 비율 높음, 적절한 차트 유형, 등장 애니메이션, 라벨 가독성 |
| **이미지/아이콘** | 10 | 일관된 스타일, 적절한 해상도, SVG 우선, 이모지 아닌 전용 아이콘 |
| **인터랙션/모션** | 10 | 의도적 전환 효과, 150~400ms 지속, prefers-reduced-motion 대응, 과하지 않음 |

**합격선**: 45/60 (75%)
**목표**: 50/60+ (83%+)
